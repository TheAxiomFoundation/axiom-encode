Add `oracle-coverage --fail-on-empty` so a coverage gate that classifies zero executable outputs fails loudly instead of passing vacuously. `is_composition_policy_repo_root` accepts a directory by name (`rulespec-<country>`) when no `.git` boundary sits at or above it, so a mis-pointed `--root` — a name-shaped non-checkout, an empty `.axiom/` directory or file, an empty pending ledger, an empty jurisdiction directory, or a spurious nested `.git` — builds a zero-output report that satisfies every `--fail-on-*` threshold and the gate goes green (axiom-encode#1113, rulespec-dk#2). `--fail-on-empty` turns a report with no executable outputs at `--root` into a non-zero exit. The decision keys off executable outputs only: program surfaces are the fixed global PolicyEngine variable manifest (independent of `--root`), so `--include-program-surfaces` must not make an empty root look non-empty. It is opt-in because a brand-new jurisdiction repo (e.g. rulespec-tz before its first module merges) is legitimately empty and its push-to-main gate must still pass, so only callers that know outputs are expected set the flag. A non-Git, ProgramSpec-only tree is accepted and classified normally — its `outputs:` yield executable outputs — not rejected as a non-checkout.
