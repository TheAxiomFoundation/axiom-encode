Waiver fingerprints are partition-invariant again: the batch fingerprint executor no longer holds a shared resolution/identity cache across modules (which leaked per-module resolution state between batch neighbors and made failure messages — and therefore fingerprints — depend on chunk layout; observed on rulespec-us as 5/3,099 fingerprints changing between partitions). Only the partition-invariant routing cache spans the batch; each module's validate opens a fresh resolution scope, so every fingerprint equals the module's standalone value under any batch composition or worker fan-out (~8% slower than the shared-cache batch).
