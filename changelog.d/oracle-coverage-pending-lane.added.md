Add a declared oracle-coverage pending lane so bulk encoding no longer needs a per-PR cross-repo oracles pin dance. A rulespec repository declares new executable outputs in a top-level `oracle-coverage-pending.yaml`; `oracle-coverage` reclassifies them from `unmapped` to `pending_classification` (visible, counted, accountable) so the gate passes, while any output declared in neither the mappings nor the pending file still fails. New `oracle-coverage-pending sync` (bulk-lane dispatcher step) and `oracle-coverage-pending check` (both-ways ratchet: undeclared-unmapped, stale declarations, ceiling) manage the file, and `oracle-coverage --fail-on-stale-pending` enforces the drain.
