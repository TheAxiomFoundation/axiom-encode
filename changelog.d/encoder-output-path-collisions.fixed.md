Fixed `_source_identifier_to_relative_rulespec_path` so corpus citations whose final segment contains a dot (CDSS MPP `63-503.132`, CA WIC `18901.10`, etc.) no longer collapse onto a section-level output path. Dotted leaves now nest as `regulations/mpp/63-503/132.yaml`. Two adjacent root-cause issues are addressed at the same time: `_run_single_eval` now derives the output path from the requested citation rather than the resolver-returned `citation_path`, so a fallback to a parent provision no longer silently relocates the encode; and `_apply_generated_encoding_result` now refuses to overwrite a target whose declared `corpus_citation_path` differs from the incoming file's.
