Extend the installed-encoder apply-identity binding with a third mode — the git-free, root-provisioned cross-repo runtime (marked by `AXIOM_ENCODE_TRUSTED_RUNTIME`) — alongside the workflow-checkout (`github-sha`) and dev (`git`) modes. The provisioner BUILDS the runtime's `axiom_encode` from a raw `ls-tree`+`cat-file` export of the attested commit (no `git archive`, so committed `.gitattributes` `export-subst`/`export-ignore` cannot inject or drop code; committed symlinks/gitlinks are refused), owning the encoder git dir root-side and reading it with hardened, worktree-free git. It verifies the commit, origin, the version all three metadata files declare, and the version-bump invariant against full history, then publishes a mode-0444 `runtime-attestation.json` — `{origin, commit, version, package_tree_sha256}`, with `O_EXCL`/`O_NOFOLLOW`. At `--apply`, `_apply_encoder_execution_identity` byte-binds the running package to `package_tree_sha256` (the same `_deterministic_tree_identity` digest main compares for the checkout case), binds the attested version to `__version__`, fails closed if more than one identity source is asserted or a live checkout conflicts, and records `identity_source` (`github-sha` / `git` / `trusted-runtime-attestation`) — required to agree across both signed manifest blocks — so consumers can distinguish the modes.
