Resolve apply-manifest placement against the checkout root, not the jurisdiction content root. `encode --apply` passes the jurisdiction content root (`<checkout>/uk`) as its policy-repo path for a country-monorepo citation, so the 0.2.1187 layout-gate check ran against `<checkout>/uk` — where the content-root prefix is empty and `.axiom/repository-structure.yaml` does not exist — and never relocated: the bulk-encode CI kept writing the un-mergeable Config-A manifest (`uk/.axiom/encoding-manifests/...`) under 0.2.1187. Placement now derives the enclosing `rulespec-*` checkout root from the content root and evaluates the layout gate there, so a country monorepo whose gate forbids `.json` under its bare-country directory (rulespec-uk/-be/-gh) correctly relocates the signed manifest to the repo-root `.axiom/encoding-manifests/<juris>/...` mirror (Config C). Regression covered by an apply test that passes the content root exactly as `cmd_encode` does (axiom-encode#1078).
