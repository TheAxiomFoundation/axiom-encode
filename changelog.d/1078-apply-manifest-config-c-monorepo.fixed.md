Place `encode --apply` signed manifests where the target repo's `repository-structure.yaml` permits them. For a country monorepo whose bare-country content root forbids `.json` (rulespec-uk/-be/-gh: `uk/**` allows only `.yaml`), the manifest is relocated from the un-mergeable `<juris>/.axiom/encoding-manifests/...` layout (Config A) to the repo-root `.axiom/encoding-manifests/<juris>/...` mirror with jurisdiction-prefixed `applied_files` (Config C) — the canonical layout that passes both the layout gate and `guard-generated`, preserving all encoder provenance (`run_id`, `backend`, `model`, `generation_prompt_sha256`, sha256s, signature, supersedes chain). Repos whose gate permits `.json` under the content root (rulespec-us, uk-regional subroots) are unchanged (axiom-encode#1078).
